# 학습 운영 시스템

이 문서는 현재 워크스페이스에서 학습 기록, 시도 결과, Q&A 개념, 추천을 어떻게 연결하는지 정리한 운영 문서다.

## 기본 원칙
- 고정 일수나 고정 시간 목표는 사용하지 않는다.
- 기록 단위는 `세션`과 `시도`를 분리한다.
- 추천은 최근 기록과 반복 혼동을 기준으로 갱신한다.
- 공식 자료 우선 원칙은 유지한다.
- 웹사이트와 주고받는 데이터는 `sync/toeic_web_sync.json` 하나로 고정한다.
- 웹사이트는 원시 기록 입력과 시각화를 담당하고, 파생 계산은 현재 프로젝트가 담당한다.

## 핵심 기록 단위

### 세션
- 뜻: 한 번의 공부 흐름 전체
- 기록 위치: `logs/sessions/`
- 남겨야 할 최소 내용:
  - 세션 ID
  - 날짜
  - 상태
  - 학습 주제
  - 사용 자료
  - 핵심 활동
  - 배운 점
  - 막힌 점
  - 다음 복습
  - 추천 반영

### 시도
- 뜻: 특정 자료, 세트, 파트, 문항군을 실제로 푼 기록
- 기록 위치: `logs/attempts/`
- 남겨야 할 최소 내용:
  - 시도 ID
  - 연결 세션 ID
  - 자료명
  - 출처
  - 파트
  - 하위능력
  - 문항 수
  - 정답 수
  - 정답률
  - 오답 유형
  - 가장 어려운 포인트
  - 시간 압박 여부
  - 다음 조치

## 기록 흐름
1. 실제 풀이가 있었다면 먼저 시도 기록을 남긴다.
2. 그다음 세션 기록에 오늘 흐름을 정리한다.
3. `tracking/progress_dashboard.md`에 최근 상태를 반영한다.
4. Q&A에서 새 개념이나 반복 혼동이 생겼으면 `tracking/qna_master_log.md`, `knowledge/concept_index.md`, `tracking/review_queue.md`를 갱신한다.
5. 자료 정책이나 추천 규칙이 바뀌면 `tracking/strategy_change_log.md`에 남긴다.
6. 웹사이트와 동기화해야 하면 마지막 승인본인 `sync/toeic_web_sync.json`을 갱신한다.

## 추천 반영 규칙
- 시도 기록이 생기면 아래를 먼저 확인한다.
  - 어떤 파트에서 틀렸는가
  - 어떤 하위능력에서 반복되는가
  - 시간 압박이 있었는가
  - 같은 개념을 Q&A에서 반복 질문했는가
- 이 결과를 `docs/04_adaptive_recommendation_model.md` 기준으로 정리해 대시보드 추천에 반영한다.

## Q&A와의 연결
- Q&A 질문은 단순 답변으로 끝내지 않는다.
- 같은 개념이 2회 이상 반복되면 개념 카드 승격 후보로 본다.
- 같은 개념이 3회 이상 반복되면 반복 혼동으로 간주하고 추천 우선순위를 올린다.

## 어휘 자산 운영
- 어휘 자산은 `마스터 목록 + 주제 세트 + 어휘 복습 큐` 구조로 관리한다.
- 주제 세트는 고정 일정표가 아니라 현재 약점, 최근 시도, 추천 근거에 따라 꺼내 쓴다.
- 주제 세트가 부족할 때만 승인된 synthetic vocab/expression set를 보충 후보로 본다.

## 업데이트 순서
1. `logs/attempts/`
2. `logs/sessions/`
3. `tracking/progress_dashboard.md`
4. 필요 시 `tracking/review_queue.md`
5. 필요 시 `tracking/strategy_change_log.md`
6. 새 사실 확인이 있었다면 `tracking/source_check_log.md`
7. 웹 반영본을 다시 내보내야 하면 `sync/toeic_web_sync.json`

## 웹 연동 원칙
- 웹사이트는 `원시 이벤트 기록기`이자 `시각화 계층`이다.
- 현재 프로젝트는 `정규화`, `약점 집계`, `추천`, `Q&A 메모리 반영`을 맡는다.
- 웹에서 받아온 파일은 먼저 revision과 event 중복 여부를 검증한다.
- 기존 원시 기록 수정은 덮어쓰기가 아니라 `raw_record.corrected` 이벤트로만 처리한다.
- v1 범위는 RC만 허용한다.
