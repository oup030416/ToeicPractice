# Q&A 스레드 운영 프로토콜

이 문서는 별도 Q&A 스레드에서 빠르게 질문하고 답을 받으면서도, 그 내용을 현재 TOEIC 프로젝트 메모리와 추천 시스템에 반영하기 위한 기준 문서다.

## 스레드 목적
- 공부 중 막히는 부분을 빠르게 묻고 바로 답을 받는다.
- 질문과 답변을 문서로 누적 저장한다.
- 반복되는 개념은 개념 카드로 승격한다.
- 저위험 학습 반영은 자동으로 복습 큐와 추천 근거에 연결한다.
- 큰 전략 수정은 자동반영하지 않고 승인 대기로 남긴다.

## 권장 스레드 이름
- `TOEIC Q&A Desk`

## 첫 지시문으로 삼을 기준
- 이 문서 전체
- `AGENTS.md`
- `docs/05_skill_system.md`

## 기본 답변 형식
1. 1단락: 핵심 답변
2. 필요 시 2단락: 짧은 이유 또는 헷갈리는 포인트
3. 필요 시 짧은 영어 예문 또는 비교
4. 마지막 한 줄: `학습 반영: ...`

## 언어 규칙
- 설명, 안내, 메타정리는 한국어
- 예문, 보기, 문제, collocation은 영어

## 저장 규칙
1. 모든 질문은 `tracking/qna_master_log.md`에 한 줄씩 누적한다.
2. 재사용 가치가 높은 개념은 `knowledge/concept_index.md`와 `knowledge/concepts/` 아래 카드로 승격한다.
3. 저위험 자동반영 항목은 `tracking/review_queue.md`에 남긴다.
4. 추천 규칙이나 자료 정책에 영향을 주는 항목은 `tracking/strategy_change_log.md`에 남긴다.
5. 질문이 웹에서 푼 RC 문항과 연결되면, 추후 `sync/toeic_web_sync.json`의 `qna.linked` 파생 이벤트로 연결할 수 있게 `qna_id`와 관련 문항 메모를 분명히 남긴다.

## 저위험 자동반영 범위
- 개념 카드 생성 또는 갱신
- 약점 태그 누적
- 다음 세션 복습 큐 추가
- 특정 파트 설명 우선순위 메모
- 추천 강도 상향 후보 표시

## 승인 필요 범위
- 공식 사실 해석 변경
- 새 자료 또는 문제집 도입
- 새 기술, 새 자동화, 새 Skill 도입
- 추천 산식 변경
- 평가 지표 변경

## 개념 카드 승격 조건
- 같은 개념 질문이 2회 이상 등장
- 여러 파트에 재사용 가능
- 다음 추천에서 다시 참조할 가치가 큼
- 현재 약점과 직접 연결됨

## 다음 추천과의 연결
- 같은 개념 질문이 3회 이상 누적되면 해당 항목을 반복 혼동 대상으로 본다.
- 반복 혼동으로 올라간 개념은 `docs/04_adaptive_recommendation_model.md`의 priority score 계산에서 가중치를 받는다.
- 전략 변경은 승인 후에만 반영한다.
